# LitPass

The on-chain identity & reputation layer for **LitVM** - Litecoin's first EVM-compatible Layer-2.

Soulbound passports, daily check-in streaks, achievement badges, real on-chain activity badges, and cross-dApp stamps. All on chain ID **4441**.

🔗 **Live:** https://litpass.xyz 
🧱 **Network:** LitVM LiteForge Testnet (chain 4441)  
📦 **Repo:** https://github.com/delreyir/litpass

## What's in the app

### 1. Soulbound passport
- Mint **one passport per wallet** as an ERC-721 (non-transferable).
- Lower token IDs are minted first - early adopters get a permanent low number.
- On-chain SVG art generated by the contract itself. No IPFS, no off-chain server.

### 2. Daily streak engine
- Single `checkIn()` call per "day" - defined by a configurable `dayLength` (1 hour on testnet, 1 day on mainnet).
- Tracks **current streak**, **longest streak**, and **lifetime check-ins**.
- Miss a day → streak resets to 1.

### 3. Streak badges (ERC-1155, soulbound)
Earned by hitting milestones on your LitPass state:

| Badge | Trigger |
| --- | --- |
| Spark | 3-day current streak |
| Flame | 7-day current streak |
| Inferno | 30-day current streak |
| Eternal | 100-day longest streak |
| Voyager | 10 check-ins |
| Pathfinder | 50 check-ins |
| Legend | 365 check-ins |

### 4. Activity badges (EIP-712 attested, soulbound)
Earned from **real on-chain history** on LitVM - no in-app actions needed. A backend service reads your activity from the Blockscout API + RPC, signs an EIP-712 attestation, and the contract verifies the signature on claim. Five families:

| Family | Badges |
| --- | --- |
| **Transactor** | Newcomer (10+ TX), Regular (20+), Power User (50+), Heavy (100+) |
| **Builder** | Builder (1+ contract deployed), Architect (3+), Shipwright (10+) |
| **Explorer** | Wanderer (5+ unique contracts touched), Pathfinder (15+), Cartographer (50+) |
| **Wallet age** | Fresh (3+ days), Settled (14+), OG (45+) |
| **Active days** | Dedicated (3+), Committed (7+), Loyalist (30+) |

### 5. Cross-dApp stamps
Any authorized issuer (other LitVM dApps) can define stamps and grant them to LitPass holders. Examples a partner could ship: "Bridged via WheelX", "Bet on MidasPredict", "Provided liquidity to Lester Labs".

### 6. Referrals
- Every profile share link carries the viewer's address as `?ref=...`.
- On first mint, the new user's referrer is bound on-chain automatically.
- Each wallet stores its referrer and referral count.

### 7. Profile pages
- Public, shareable URL: `/p/[address]`.
- Server-rendered with on-chain data: passport, streak, badges, stamps, referrals.
- **Dynamic OG image** at `/p/[address]/opengraph-image` - Twitter, Telegram, Warpcast all show a rich preview with your streak and stats.
- Share buttons for X, Warpcast, and native mobile share.

### 8. Leaderboard
- `/leaderboard` ranks every wallet by total check-ins and current streak.
- Indexes `CheckedIn` events on demand via viem (`getContractEvents`), starting from the deployment block to avoid RPC timeouts.
- Cached for 30 s at the edge.

│   ├── badges/                   Tabbed: Activity | Streak

### EIP-712 attestation flow (activity badges)

```
┌──────────┐  POST /api/attest   ┌──────────┐
│  Frontend│ ───────────────────▶│ Backend  │
│ (wagmi)  │                     │ (Node)   │
└─────┬────┘                     └────┬─────┘
      │                               │ 1. Read on-chain history
      │                               │    via Blockscout v2 + viem
      │                               │
      │     signature + nonce         │ 2. Sign EIP-712 claim with
      │ ◀─────────────────────────────│    signer private key
      │                               │
      ▼
┌──────────────────────┐
│ ActivityBadges.claim │  3. Contract recovers signer from sig
│       (on-chain)     │     and verifies it matches the trusted signer
└──────────────────────┘  4. If measuredValue >= threshold → mints badge
```

The backend's signer wallet only signs off-chain messages. It never holds funds and never sends transactions, so leaking its key would let an attacker fake attestations but not steal anything. The contract owner can rotate the signer at any time with `setSigner(newSigner)`.

## Running locally

### Contracts
```bash
cd contracts
cp .env.example .env            # add PRIVATE_KEY and ATTESTATION_SIGNER_ADDRESS
npm install
npx hardhat compile
npx hardhat test                # 21 tests
npx hardhat run scripts/deploy.ts --network litvmTestnet
```

Deploy script writes addresses to:
- `contracts/deployments/litvmTestnet.json`
- `web/lib/deployment.json` (auto-mirrored)

### Frontend
```bash
cd web
npm install
echo "ATTESTATION_SIGNER_KEY=0x..." > .env.local
npm run dev
```

Open http://localhost:3000.

## What makes LitPass different

- **First identity layer on LitVM** - no other native passport / reputation contract on LiteForge today.
- **Activity-based reputation** - most "soulbound passport" projects elsewhere reward only in-app streaks. LitPass mints badges based on real wallet history, which is censorship-resistant and Sybil-resistant.
- **Composable** - any dApp can read a wallet's badges and stamps on-chain. The `StampRegistry` is open: register as an issuer and grant your own stamps.
- **No off-chain dependencies in core flows** - the passport SVG is rendered on-chain, the streak engine is on-chain, badges are on-chain. The backend attestor is only used for activity badges, where reading historical chain data from a smart contract isn't possible.
